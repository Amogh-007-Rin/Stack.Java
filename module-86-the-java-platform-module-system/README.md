# Module 86: The Java Platform Module System (JPMS)

## Overview
The Java Platform Module System (JPMS), introduced in **Java 9** (Project Jigsaw), provides a way to modularize Java applications. A module is a named, self-describing collection of code and data, with explicitly declared dependencies and public API surfaces.

## Learning Objectives
- Understand the structure of `module-info.java`
- Use `requires`, `exports`, and `opens` directives
- Distinguish named modules from unnamed and automatic modules
- Compile and run a multi-module application using `--module-source-path`

## Prerequisites
- Familiarity with packages and JAR files
- Basic command-line compilation with `javac`

## Theory

### module-info.java
Every named module has a `module-info.java` declaration file at its root:

```java
// src/com.example.greeter/module-info.java
module com.example.greeter {
    exports com.example.greeter;
}
```

### Key Directives

| Directive | Purpose |
|-----------|---------|
| `requires` | Declares a dependency on another module |
| `requires transitive` | Makes the dependency transitive to consumers |
| `exports` | Makes a package accessible to other modules |
| `exports ... to ...` | Qualified export ‚Äî only to listed modules |
| `opens` | Allows reflection access to a package (at runtime) |
| `opens ... to ...` | Qualified open |
| `provides ... with ...` | Service provider declaration |
| `uses` | Service consumer declaration |

### Module Types

| Module Type | Description |
|-------------|-------------|
| **Named module** | Has a `module-info.java`; explicitly declares dependencies and exports |
| **Unnamed module** | Code on the classpath; reads all other modules; all packages are exported |
| **Automatic module** | A JAR on the module path without `module-info.java`; its name is derived from the JAR filename |

### Compiling Multi-Module Projects

```bash
# Directory structure:
# src/
#   com.greeter/
#     module-info.java
#     com/greeter/Greeter.java
#   com.app/
#     module-info.java
#     com/app/Main.java

# Compile
javac --module-source-path src -d out -m com.greeter,com.app

# Run
java --module-path out -m com.app/com.app.Main
```

### Why Modules?
- **Strong encapsulation** ‚Äî internal packages are hidden by default
- **Explicit dependencies** ‚Äî no more classpath spaghetti
- **Reliable configuration** ‚Äî missing modules are detected at startup, not runtime
- **Scalability** ‚Äî only necessary modules are loaded; improved startup time for large apps

## Common Pitfalls
- **Forgetting to export packages** ‚Äî if a package is not exported, other modules cannot access its public types
- **Split packages** ‚Äî the same package cannot be in two modules on the module path
- **Reflection without `opens`** ‚Äî reflection on non-exported or non-open packages fails at runtime
- **Automatic module name guessing** ‚Äî JARs without `module-info.java` get module names from their filenames, which can be unpredictable
- **`requires` vs `requires transitive`** ‚Äî use `requires transitive` when re-exporting dependencies

## Quiz

1. Which directive makes a package accessible to other modules?
   a) `requires`
   b) `exports`
   c) `opens`
   d) `provides`

2. What is the default access of a package in a named module?
   a) Accessible to all modules
   b) Accessible only within the module
   c) Accessible only on the classpath
   d) Accessible to automatic modules only

3. How do you allow another module to use reflection on your module's package?
   a) `exports`
   b) `requires`
   c) `opens`
   d) `transitive`

4. What is an automatic module?
   a) A module generated by the compiler
   b) A JAR on the module path without a module-info.java
   c) A module that requires no other modules
   d) A module that exports all packages

5. Which of the following is NOT a benefit of JPMS?
   a) Strong encapsulation
   b) Explicit dependency declarations
   c) Automatic memory management
   d) Reliable configuration and startup detection of missing modules

---

**Answers:** 1-b, 2-b, 3-c, 4-b, 5-c

## Key Takeaways
- `module-info.java` defines a module's dependencies and exported packages
- `exports` controls compile-time and runtime access; `opens` controls reflection access
- Named modules have strong encapsulation ‚Äî unexported packages are inaccessible
- Compile multi-module projects with `--module-source-path` and `-m`
- JPMS improves application reliability, security, and startup time
