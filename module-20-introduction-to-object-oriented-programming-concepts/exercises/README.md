# Module 20 — Exercises

## Exercise 1: Procedural → OOP Thinking
Think of a real-world process you interact with daily (e.g., a coffee machine, a bank account, a vending machine). Write down:
- What objects are involved?
- What state (fields) does each object have?
- What behavior (methods) does each object have?

**Bonus:** Draw a simple class diagram showing the relationship between the objects.

## Exercise 2: Procedural vs OOP — Grocery Store
Imagine you are writing software for a grocery store. Describe how you might model a `Product` and a `ShoppingCart`:
- In a **procedural** style (using separate arrays and functions)
- In an **OOP** style (using classes with fields and methods)

Write a short paragraph comparing the two approaches.

## Exercise 3: Identify the OOP Pillar
For each scenario below, identify which OOP pillar (Encapsulation, Inheritance, Polymorphism, or Abstraction) is being applied:
1. A `Vehicle` class with `start()` method; `Car` and `Motorcycle` provide different implementations.
2. A `BankAccount` class keeps the `balance` field private and provides `deposit()` and `withdraw()` methods.
3. A `Dog` class extends `Animal` and inherits its `eat()` method.
4. A `Payment` interface with a `pay()` method; `CreditCardPayment` and `PayPalPayment` implement it differently.
